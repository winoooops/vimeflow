VIBM — 4 Screen Prompts for Google Stitch

Each prompt includes the shared preamble (not shown here) plus screen-specific details below

Screen 1 of 4
Prompt 1: Main Chat View

Screen: Main Chat View — the primary screen of the application.

Left icon rail: Vertical strip of circular project avatars/icons. Active project has a pill-shaped highlight. A "+" button at the bottom to add new projects. Small dot indicators for unread activity.

Sidebar (260px): Top section has a search bar with keyboard shortcut hint (⌘K). Below that, conversation groups displayed as collapsible sections — like Discord channel categories or forum sections. Each group header shows a name and count. Inside each group, conversations listed with: agent avatar/icon, conversation title, last message preview (truncated), timestamp, and unread badge. Active conversation has a highlighted background. Support for subtopics/threads — show a few threaded replies indented under a parent conversation.

Main area: Tab bar at top with 4 tabs: Chat (active, underlined), Files, Editor, Diff. Below the tabs, a chat stream showing a realistic coding conversation between a user and an AI coding agent. Include:

  • User message bubbles (right-aligned or left with user avatar)
  • Agent response with a code block (syntax-highlighted, Catppuccin theme)
  • An inline file change card showing filename + lines changed
  • A "thinking/reasoning" collapsed section the user can expand
  • Agent status indicator (typing... or context window usage bar)

Bottom: Message input bar with rich formatting options, attach file button, and a send button. Hint text: "Ask your agent..."

Right panel (optional, 280px): Agent status card showing: agent name, model being used, context window usage (progress bar), conversation token count, and a list of active tools/actions.

Screen 2 of 4
Prompt 2: File Explorer Tab

Screen: File Explorer Tab — same layout as chat view but the Files tab is active.

Sidebar: Identical to chat view (always persistent).

Main area: Tab bar with Files tab now active/underlined. Below, a full file tree explorer similar to VS Code's sidebar but taking the full main content width. Show:

  • Project root folder expanded with realistic file structure (src/, components/, utils/, tests/)
  • File icons by type (TypeScript, Rust, JSON, Markdown, etc.)
  • Git status indicators next to modified/added/deleted files (colored dots or letters: M, A, D)
  • Drag-and-drop visual affordance — show one file mid-drag with a drop target highlighted
  • Right-click context menu floating near a file (Rename, Delete, Copy Path, Open in Editor, View Diff)
  • A breadcrumb path bar at the top of the file tree

Bottom bar: File count, total size, and a "Drop files here to add" zone.

Screen 3 of 4
Prompt 3: Code Editor Tab

Screen: Code Editor Tab — the Editor tab is active.

Sidebar: Identical to other views (always persistent).

Main area: Tab bar with Editor tab active. Below, a code editor view inspired by vim/emacs style with Catppuccin syntax highlighting. Show:

  • Open file tabs along the top of the editor area (like VS Code file tabs) — 2-3 files open, one active
  • Line numbers in the gutter (subtle color)
  • Syntax-highlighted TypeScript/React code using Catppuccin Mocha colors
  • A vim/emacs mode indicator in the bottom status bar (e.g., "-- NORMAL --" or "-- INSERT --")
  • Cursor visible on a line with a subtle highlight on the active line
  • A minimap on the right edge (like VS Code)
  • Bottom status bar showing: file name, line:column, encoding, language mode, vim mode

Key detail: The editor should feel integrated, not like a separate app. Same frosted glass panels, same Catppuccin palette, same rounded corners on containers.

Prompt 4: Diff View Tab

Screen: Git Diff View Tab — the Diff tab is active.

Sidebar: Identical to other views (always persistent).

Main area: Tab bar with Diff tab active. Below, a split-pane diff viewer showing code changes. Show:

  • Side-by-side diff layout (left = before, right = after)
  • Added lines highlighted with subtle green background (#a6e3a1 at 15% opacity)
  • Removed lines highlighted with subtle red background (#f38ba8 at 15% opacity)
  • Modified lines with a changed-word highlight (inline diff)
  • A file selector dropdown or list at the top showing changed files with +/- line counts
  • Navigation arrows to jump between hunks/changes
  • Commit info bar at top: commit hash, author, message, timestamp
  • An "Accept Changes" or "Stage" button group for interactive staging

Key detail: Diff colors should harmonize with the Catppuccin palette — not the harsh default red/green but the softer Catppuccin equivalents at low opacity.
