Google Stitch Prompt Strategy

Three approaches to get the best UI output from Stitch

A

Single Comprehensive Prompt

One detailed prompt describing the entire app — all 4 tabs, sidebar, theme, and interactions. Stitch generates everything in one shot.

Pros

  • Consistent visual language across all views
  • Single generation = unified design system
  • Fastest to execute

Cons

  • May lose detail on individual screens
  • Risk of generic output if prompt is too broad
  • Hard to iterate on specific views
B

Screen-by-Screen Prompts (Recommended)

Four focused prompts — one per tab view (Chat, Files, Editor, Diff). Each prompt shares a common "design system" preamble for consistency, then adds screen-specific details.

Pros

  • Maximum detail per screen
  • Can iterate on individual views without redoing everything
  • Shared preamble keeps visual consistency
  • Best control over each tab's UX

Cons

  • More prompts to manage (4-5 total)
  • Minor visual drift between screens possible
C

Progressive Refinement

Start with a broad "design system + main chat" prompt. Use Stitch's follow-up capability to generate each additional tab iteratively, refining as you go.

Pros

  • Most adaptive — refine based on what Stitch produces
  • Builds on established visual language
  • Good if Stitch supports multi-turn

Cons

  • Depends on Stitch's multi-turn capabilities
  • Earlier decisions constrain later screens
  • Takes longest to execute
