Prompt 0 — Generate First
General App Overview

What is Vimeflow: Vimeflow (Vibe + Vim + Flow) is a desktop application for managing conversations with AI coding agents — a developer's command center that combines chat, file management, code editing, and git diffs in one unified interface. Built around vim-style navigation and designed to keep developers in a flow state.

Screen: Full application overview — the general view showing the complete layout structure of Vimeflow.

Purpose: Establish the spatial layout, visual hierarchy, and design language for the entire application. This is the foundational reference for all other screens.

Layout (left to right):

  • Icon rail (48px, leftmost): Vertical strip of circular project/workspace icons. One is active with a pill-shaped highlight. A "+" icon at the bottom. Small colored dot indicators for unread activity. The rail has a slightly darker or more opaque glass background than the rest.
  • Sidebar (260px): Frosted glass panel. Top: search bar with ⌘K hint. Below: auto-grouped conversation list organized like Discord channel categories or forum sections. Group headers are collapsible. Each conversation item shows: small agent icon, title, preview text, timestamp, unread badge. One conversation is visually selected/active. A few sub-threads shown indented under one conversation.
  • Main content area (flexible): Tab bar along the top with 4 tabs: Chat, Files, Editor, Diff. The Chat tab is active. Below the tab bar, show a partially visible chat conversation with 2-3 messages — enough to convey the content type but the focus is on the overall structure, not chat details.
  • Right panel (280px, optional): Agent status card showing agent name, model indicator, context window usage as a progress bar, and a small list of recent actions/tools used. This panel should feel secondary — lighter glass effect or slightly recessed.

Visual emphasis:

  • Frosted glass / transparency effect on all panels — the desktop wallpaper or a subtle gradient should be faintly visible through the surfaces
  • Clear visual separation between the 4 columns (rail, sidebar, main, right panel) using subtle borders or shadow edges, not hard lines
  • The window itself should have rounded corners and a native title bar or custom traffic lights (macOS style)
  • Overall feeling: professional, warm, developer-friendly — not sterile or corporate

Command palette overlay: Show the vim-style command palette active on screen — a floating dialog centered horizontally, positioned in the upper third of the window. It has a text input prefixed by : with :open typed, and 4-5 fuzzy-matched suggestion rows below it (e.g., :open App.tsx, :open utils.ts, :open-recent, :open-project). Each row has an icon, command name, and a keyboard shortcut hint on the right. The palette has a frosted glass background with the Catppuccin surface color and a subtle border glow. The rest of the app is slightly dimmed behind it.

Mood references: The spatial density of Discord, the warmth of Obsidian's dark theme, the polish of OpenAI Desktop, all unified under Catppuccin Mocha palette with frosted glass surfaces.

  • Agent response containing a syntax-highlighted code block (Catppuccin Mocha theme)

Prompt 2
Files Tab (Detail)

Screen: File Explorer Tab — Files tab active in the tab bar.

Sidebar: Same as overview — persistent.

Main area: Full file tree explorer taking the main content width. Show:

  • Project root expanded with realistic structure: src/, components/, utils/, tests/, package.json, tsconfig.json
  • File type icons (TypeScript .ts, Rust .rs, JSON, Markdown, folders)
  • Git status indicators: colored dots or letters (M=modified, A=added, D=deleted) next to affected files
  • One file shown mid-drag with a blue drop-target highlight on a folder
  • A floating right-click context menu near one file: Rename, Delete, Copy Path, Open in Editor, View Diff
  • Breadcrumb path bar at the top: project-name / src / components /

Bottom bar: File statistics (count, total size) and a subtle "Drop files here to add" drop zone.

Prompt 3
Editor Tab (Detail)

Screen: Code Editor Tab — Editor tab active.

Sidebar: Same as overview — persistent.

Main area: Integrated code editor with vim/emacs style, Catppuccin syntax highlighting:

  • Open file tabs at top of editor area — 2-3 files open (e.g., App.tsx, utils.ts, types.ts), one active
  • Line numbers in a subtle gutter
  • Syntax-highlighted TypeScript/React code using Catppuccin Mocha colors
  • Active line highlighted with a subtle background tint
  • Cursor visible, block-style (vim normal mode)
  • Minimap on the right edge
  • Bottom status bar: filename, Ln 42 Col 18, UTF-8, TypeScript, "-- NORMAL --" vim mode indicator

Key: Editor feels integrated into the app — same glass panels, Catppuccin palette, rounded container edges. Not a separate embedded widget.

Prompt 4
Diff Tab (Detail)

Screen: Git Diff View Tab — Diff tab active.

Sidebar: Same as overview — persistent.

Main area: Side-by-side diff viewer for code changes:

  • Split pane: left = before, right = after
  • Added lines: subtle green background (Catppuccin green #a6e3a1 at ~15% opacity)
  • Removed lines: subtle red background (Catppuccin red #f38ba8 at ~15% opacity)
  • Modified words highlighted inline within changed lines
  • File selector bar at top: list of changed files with +/- line counts (e.g., "src/App.tsx +12 -3")
  • Navigation arrows (↑↓) to jump between diff hunks
  • Commit info bar: short hash, author avatar, commit message, relative timestamp
  • "Stage Hunk" / "Discard" button group for interactive staging

Key: Diff colors use soft Catppuccin-tinted backgrounds, not harsh default red/green. Everything harmonizes with the overall warm palette.
