# ts-rs Type Codegen — Single Source of Truth for Rust↔TS Contracts

**Date:** 2026-04-11
**Status:** Proposed
**Motivation:** PR #47 review found 5 type-contract mismatches between
Rust serde output and TypeScript interfaces. All were caused by manual
duplication — developers wrote Rust structs without checking TS types.
Automated codegen eliminates this class of bug entirely.

## Approach

Use [`ts-rs`](https://github.com/Aleph-Alpha/ts-rs) to derive TypeScript
interfaces from Rust structs. ts-rs reads `serde` attributes
(`rename_all`, `skip_serializing_if`, etc.) and produces correct TS.

## Setup

### 1. Add dependency

```toml
# src-tauri/Cargo.toml
[dev-dependencies]
ts-rs = "10"
```

`dev-dependencies` because codegen runs via `cargo test`, not at runtime.

### 2. Annotate structs

Add `#[derive(TS)]` and `#[ts(export)]` to every struct/enum that
crosses the Tauri IPC boundary:

```rust
use ts_rs::TS;

#[derive(Debug, Clone, Serialize, TS)]
#[ts(export, export_to = "../../src/bindings/")]
#[serde(rename_all = "camelCase")]
pub struct FileDiff {
    pub file_path: String,
    pub old_path: Option<String>,
    pub new_path: Option<String>,
    pub hunks: Vec<DiffHunk>,
}
```

ts-rs respects `serde(rename_all = "camelCase")`, so `file_path` →
`filePath` in the generated TS — no manual mapping needed.

### 3. Generate bindings

```bash
cargo test        # generates src/bindings/*.ts
```

Each `#[ts(export)]` struct produces a file like
`src/bindings/FileDiff.ts`:

```ts
// This file was generated by [ts-rs](https://github.com/Aleph-Alpha/ts-rs). Do not edit this file manually.

export interface FileDiff {
  filePath: string
  oldPath: string | null
  newPath: string | null
  hunks: DiffHunk[]
}
```

### 4. Frontend imports from bindings

Replace hand-written interfaces with imports from the generated files:

```ts
// Before (manual, error-prone)
export interface FileDiff {
  filePath: string
  oldPath?: string
  // ... might drift from Rust
}

// After (generated, always correct)
export type { FileDiff } from '../bindings/FileDiff'
```

### 5. CI verification

Add a CI step that regenerates bindings and fails if they differ from
what's committed:

```yaml
- name: Verify ts-rs bindings are up to date
  run: |
    cargo test
    git diff --exit-code src/bindings/
```

This catches PRs where someone changed a Rust struct but forgot to
regenerate bindings.

## Structs to annotate

All IPC-facing types in `src-tauri/src/`:

| Module                 | Structs/Enums                                                                          |
| ---------------------- | -------------------------------------------------------------------------------------- |
| `git/mod.rs`           | `ChangedFileStatus`, `ChangedFile`, `DiffLineType`, `DiffLine`, `DiffHunk`, `FileDiff` |
| `filesystem/types.rs`  | `FileEntry`, `ReadFileRequest`, `WriteFileRequest` (if they exist)                     |
| `terminal/commands.rs` | `SpawnPtyRequest`, `WritePtyRequest`, etc.                                             |

## Migration plan

1. Add `ts-rs` to `Cargo.toml` dev-dependencies
2. Add `#[derive(TS)]` + `#[ts(export)]` to git module types first
3. Run `cargo test` → generates `src/bindings/`
4. Replace hand-written types in `src/features/diff/types/index.ts` with
   re-exports from `src/bindings/`
5. Delete the now-redundant manual interface definitions
6. Repeat for filesystem and terminal types
7. Add CI check
8. Add `src/bindings/` to `.gitignore` OR commit them (trade-off below)

### Commit vs gitignore generated bindings

| Approach      | Pro                                                                  | Con                                                                    |
| ------------- | -------------------------------------------------------------------- | ---------------------------------------------------------------------- |
| **Commit**    | Frontend devs don't need Rust toolchain; CI diff check catches drift | Generated files in git history                                         |
| **Gitignore** | Clean history                                                        | Requires `cargo test` before any frontend work; CI must generate first |

**Recommendation:** Commit them. The CI diff check is the safety net, and
frontend-only contributors shouldn't need `cargo` installed.

## `Option<T>` mapping

ts-rs maps `Option<String>` to `string | null` by default. The existing
TS interfaces use `?: string` (optional). To match:

```rust
#[ts(optional)]  // generates `fieldName?: string` instead of `fieldName: string | null`
pub old_path: Option<String>,
```

Or configure globally via `ts-rs` features. Check docs for the version
used.

## Out of scope

- **tauri-specta** (typesafe invoke wrappers) — consider as a follow-up
  once ts-rs is established. Adds typesafe command calling but requires
  more setup.
- **JSON Schema** — heavier, less ergonomic for Tauri apps.
- **Generating Rust from TS** — ts-rs is one-directional (Rust → TS).
  Rust is the source of truth since it's the backend.
