## Statusline Script Generation

When `spawn_pty` is called, the Rust backend:

1. Creates session directory: `<project>/.vimeflow/sessions/<session-id>/`
2. Generates a statusline script at `<project>/.vimeflow/sessions/<session-id>/statusline.sh`:

```bash
#!/usr/bin/env bash
# Auto-generated by Vimeflow for session <session-id>
# Receives Claude Code statusline JSON on stdin, writes to status file
cat > "<project>/.vimeflow/sessions/<session-id>/status.json"
```

3. Makes the script executable: `chmod +x`
4. Generates a settings overlay file at `<project>/.vimeflow/sessions/<session-id>/settings.json`:

```json
{
  "statusLine": {
    "type": "command",
    "command": "<project>/.vimeflow/sessions/<session-id>/statusline.sh",
    "refreshInterval": 5
  }
}
```

### How the settings are injected

When the user launches Claude Code in the terminal, Vimeflow can't force CLI flags. Instead:

**Option A — Environment variable**: Set `CLAUDE_SETTINGS_FILE=<path>/settings.json` in the PTY's environment when spawning. This is the least invasive approach if Claude Code supports it.

**Option B — Config directory overlay**: Set `CLAUDE_CONFIG_DIR` to a directory containing a `settings.json` that merges with the user's existing settings. Risk: may override user's own settings.

**Option C — Don't auto-inject**: Require the user to manually configure their statusline (documented in README). The watcher still works — it just watches for the file to appear.

**Recommended: Option A with fallback to C.** Try env injection. If Claude Code doesn't support `CLAUDE_SETTINGS_FILE`, document manual setup and detect the status file when it appears.

### SpawnPtyRequest changes

Add optional field:

```rust
pub struct SpawnPtyRequest {
    // ... existing fields ...
    pub agent_status_dir: Option<String>,  // If set, generate statusline bridge files
}
```

When `agent_status_dir` is provided, the spawn command generates the statusline files and adds environment variables to the PTY process.

## Detection → Watcher Pipeline

The frontend `useAgentStatus` hook drives the lifecycle:

```
1. Session becomes active
2. Frontend polls `detect_agent_in_session(sessionId)` every 2s
3. Agent detected → call `start_agent_watcher(sessionId, statusFilePath)`
4. Watcher receives first statusline → extracts transcript_path
5. Watcher calls `start_transcript_watcher(sessionId, transcriptPath)` internally
6. Both watchers emit Tauri events → frontend receives and updates state
7. Detection poll continues to detect agent exit
8. Agent exits → frontend calls `stop_agent_watcher(sessionId)` → watchers cleaned up
9. Panel shows final state for 5s, then collapses
```

### Session cleanup

When a session is removed (`removeSession` in `useSessionManager`):

1. Stop agent watcher
2. Stop transcript watcher
3. Optionally clean up `.vimeflow/sessions/<id>/` directory

When the app closes:

1. All watchers are stopped (Tauri's cleanup hooks)
2. `.vimeflow/sessions/` directory can be cleaned up on next launch

## Frontend Integration

### useAgentStatus modifications

Add detection polling:

```typescript
// Inside useAgentStatus:
useEffect(() => {
  if (!sessionId) return

  const interval = setInterval(async () => {
    const result = await invoke('detect_agent_in_session', { sessionId })
    if (result && !isActive) {
      setIsActive(true)
      setAgentType(result.agentType)
      // Start watcher
      const statusPath = `${projectDir}/.vimeflow/sessions/${sessionId}/status.json`
      await invoke('start_agent_watcher', {
        sessionId,
        statusFilePath: statusPath,
      })
    } else if (!result && isActive) {
      // Agent exited
      await invoke('stop_agent_watcher', { sessionId })
      // Hold state for 5s before collapsing
      setTimeout(() => setIsActive(false), 5000)
    }
  }, 2000)

  return () => clearInterval(interval)
}, [sessionId, isActive])
```

## Notes

- This spec deliberately has an "Option A with fallback to C" approach for settings injection. During implementation, test whether `CLAUDE_SETTINGS_FILE` or `--settings` flag works with Claude Code's interactive mode. If neither works, implement Option C (manual setup with docs).
- The 5-second "hold final state" before collapse prevents jarring UI when an agent briefly disconnects (e.g., during a restart).
- The `.vimeflow/` directory is session-scoped and ephemeral — it's not project config. Old session directories can be cleaned up periodically.
