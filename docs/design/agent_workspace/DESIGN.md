# Agent Workspace — Screen Design Notes

> This screen follows the unified design system in [docs/design/DESIGN.md](../DESIGN.md). This file documents screen-specific patterns and token reconciliation notes.

## Overview

The Agent Workspace is the primary application view — a CLI coding agent control plane with terminal panes, session management, and real-time agent activity monitoring. It replaces the previous Chat view as the main screen.

## Screen Zones

See root `DESIGN.md` → "App Layout (4-Zone, Discord Pattern)" for the authoritative layout spec.

1. **Icon Rail** — Project selector
2. **Sidebar** — Session list + context switcher (Files/Editor/Diff)
3. **Terminal Zone** — Tabbed agent PTY + shell panes
4. **Agent Activity** — Status, context, usage, collapsible file/tool/test sections

## Screen-Specific Patterns

### Terminal Prompt Style

The Stitch mockup uses a custom prompt format:
```
obsidian-cli ~/workspace/project-ts main*
```

In implementation, this will be replaced by the actual shell prompt from the PTY. The terminal renders whatever the agent process outputs.

### Inline Code Suggestions

The terminal mockup shows an inline "Suggested Change" card with diff highlighting. This is a visual treatment for when the terminal parser detects Claude Code's code suggestion output:
- Background: `surface-container-low/50`
- Rounded: `xl`
- Line numbers in `primary-container`
- Removed lines in `error-dim`, added lines in `success-muted`

### Agent Activity Footer

The mockup includes a decorative "waiting for command..." shell prompt in the footer. This is a design flourish — in implementation, it becomes a mini command input for quick actions (optional, not MVP).

## Design Philosophy

The Stitch-generated DESIGN.md introduced the name **"The Obsidian Terminal"** and subtitle **"The Ethereal Command."** These are expressions of the overarching Obsidian Lens design system, not replacements. The core philosophy of "Intentional Depth" — stacked sheets of obsidian glass floating in a nebula — applies equally to the terminal workspace.

Key principles carried forward:
- **Borderless architecture** — tonal shifts define space
- **Editorial developer** — Manrope headlines, Inter body, JetBrains Mono for "The Work"
- **Elevation by Tone** — brighter = higher, not shadowier
- **The Lens Blur** — modals blur and hue-shift the background
