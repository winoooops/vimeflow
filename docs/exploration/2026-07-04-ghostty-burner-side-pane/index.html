Ghostty Burner Side Pane Options

The burner should stop being a floating terminal dialog. The safest route is to make it a normal Vimeflow shell pane rendered by the existing native Ghostty surface path.

RECOMMENDED: SIDE PANE
vimeflow ghostty canvas

burner ~/vimeflow
ephemeral shell, no agent bridge, rendered by Ghostty

Mechanism

1 Clicking burner creates or reveals a Vimeflow shell pane.
2 The pane uses an ephemeral PTY and disables the agent bridge.
3 Ghostty gets no special split command; it just renders another native surface for another React pane.
4 The host header gets a small burner cue. The burner header gets the temporary identity and cwd sync control.

A. Vimeflow Side Pane

Best v1. Reuses split layout, pane lifecycle, Ghostty native rendering, shortcuts, and xterm fallback.

B. Overlay Terminal

Avoid. It revives the floating-terminal problem and makes focus, shutdown, and resize harder than a pane.

C. Ghostty Internal Split

Blocked for now. Our bridge exposes one TerminalView per surface, not a host-controlled split tree.
